📐 디자인 리서치: 영상 디제스트 UI

2026-06-11 · 드로어 vs 페이지 · 버튼 병행 vs 통합 · 웹 리서치 기반(Lazyweb 스크린샷 DB 미설치)

TL;DR — 선두 제품 전부(YouTube 자체, Readwise Reader, Glasp, Eightify, Snipd)가 트랜스크립트·요약·인사이트를 "재생 중인 플레이어 옆"에 배치합니다. 분석 화면을 재생과 분리한 제품은 사실상 없습니다. 결론: 드로어형 + 라디오 플레이어 결합(제3안)과 버튼 병행 유지가 정답에 가깝습니다.

✅ 권고

1. 결정 A → 드로어형 + 플레이어 결합 (A-① 개선판)

  • Smashing Magazine 결정 트리: 반복 진입 + 배경 컨텍스트 유지가 필요한 고볼륨 보조 콘텐츠 = 드로어
  • Eightify·Glasp 모두 유튜브 옆 사이드바로 성공 — 별도 페이지로 보내는 제품 없음
  • 이미 있는 FloatingRadioPlayer에 영상을 로드하면 "유튜브 트랜스크립트 패널" 경험이 공짜로 완성

2. 결정 B → 병행 유지 (B-①)

  • Eightify는 깊이별 모드(short/insightful/Q&A)를 따로 제공 — "가볍게 vs 깊게"는 검증된 니즈 구분
  • 비용 구조와 일치: 3줄 요약 1콜 vs 디제스트 5~7콜
  • 단, 디제스트 드로어 첫 화면에 기존 3줄 요약을 재사용 표시해 일체감 부여

3. 흡수할 디테일

  • 오토스크롤 트랜스크립트 (Readwise) — 재생 위치 따라 자막이 텔레프롬프터처럼 이동
  • 재생 위치 → 디제스트 역동기화 — 현재 구간의 섹션·인사이트 하이라이트
  • 자막 줄 전체 클릭 = 시킹 (YouTube/Snipd/Readwise 공통)
  • "이 영상에 질문" (YouTube Ask 패턴) — FeedQADrawer 인프라 재사용, Phase 2
  • 인용 → 공유 카드 (Snipd) — Phase 2 선택

🚫 안티패턴

  • 재생과 분리된 분석 전용 화면 — 타임스탬프가 죽은 텍스트가 됨
  • 경량 경로 제거 — Snipd 리뷰 주요 불만이 "AI 과개입·통제감 상실". 1콜짜리 빠른 요약은 남길 것
  • 자동 생성 남발 — 명시적 버튼 + 캐시 원칙 유지

💎 눈에 띈 디테일

  • Readwise Reader의 "비디오=문서" 관점 — 하이라이트가 지식 시스템에 쌓이는 구조, 사용자의 노션 리소스/서머리 루프와 동일 철학
  • Snipd의 순간 캡처 — 듣다가 트리플탭으로 그 순간 스니펫화 → "지금 구간 북마크" Phase 2 후보
  • YouTube Ask의 pill 버튼 — AI 기능을 메뉴 속이 아닌 플레이어 바로 아래 1차 동선에
