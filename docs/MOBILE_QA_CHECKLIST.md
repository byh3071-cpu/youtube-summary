# 모바일·PWA 수동 QA 체크리스트

자동화(E2E)로 대체하기 어려운 **실기기·제스처** 위주. 릴리스 전 샘플 기기에서 한 번씩 확인한다.

## 공통

| # | 항목 | 기대 |
|---|------|------|
| 1 | 홈 피드 스크롤 | 세로 스크롤만 동작, 가로 스와이프로 URL이 바뀌지 않음 |
| 2 | 키워드 필터 가로 스크롤 | 필터 칩 영역만 가로 스크롤 |
| 3 | 모바일 메뉴 드로어 | 열림/닫힘, `Escape`, 오버레이 탭으로 닫힘, 스크롤 격리 |
| 4 | `/trends` | 헤딩·워드 영역·빈 상태 문구 중 하나 표시 |
| 5 | 피드 Q&A FAB | 라디오 바와 겹치지 않게 탭 가능, 패널 스크롤 |

## Safari (iOS)

| # | 항목 | 기대 |
|---|------|------|
| 6 | 로그인(OAuth) | 리다이렉트·세션 유지 |
| 7 | PWA 홈 화면 추가 | 아이콘(정사각 192/512)·전체 화면 기동, 가로 회전 시 깨짐 없음 |
| 8 | 라디오 재생 | 백그라운드 전환 시 정책에 맞게 동작(무음 방지 등) |
| 8a | PWA 업데이트 | 새 배포 후 재방문 시 ChunkLoadError 없이 최신 화면 (HTML 미캐시 정책) |
| 8b | 오프라인 | 비행기 모드에서 `/offline.html` 안내 표시, 복구 후 "다시 시도" 동작 |

## Chrome (Android)

| # | 항목 | 기대 |
|---|------|------|
| 9 | 하단 safe-area | FAB·라디오가 홈 인디케이터에 가리지 않음 |
| 10 | 테마 | 시스템 다크 전환 시 레이아웃 깨짐 없음. 메뉴의 **테마 전환** 버튼으로 라이트/다크 전환 후 홈·카드·Q&A·라디오 대비 확인 |
| 11 | 빈 라디오 안내 | 큐가 빈 상태에서 모바일 안내 배너 **닫기** 동작, 닫은 뒤 피드/Q&A 가림 없음 |
| 12 | 44px 터치 | 로그인·새로고침·필터 열기·보기 전환·더 보기 버튼이 한 번에 탭됨 (실기기) |
| 13 | 결제(Stripe) | `/pricing` 결제 흐름 — 자동화하지 않음, 테스트 모드로 수동 확인 |
