# 2026-06-24 · Focus Feed 리디자인 브레인스토밍

## 무엇
"채널 추가가 불편" 질문에서 출발 → 진짜 문제는 **"FF를 안 쓰게 되는 이유"** 임이 드러남.
브레인스토밍으로 제품 정체성·디자인 방향을 재정의하고, 4축 법적 리서치 + 19개 디자인 레퍼런스를 수집.

## 결론 (요지)
- **정체성:** 발견(끌리는 영상) + 지식 워크플로 중간다리 + 쉬운 캡처. "또 다른 유튜브" ❌.
- **디자인 북극성:** "유튜브 콘텐츠 × 스포티파이 껍데기" — 영상을 음악앱처럼 틀고·큐담고·관리. 스포티파이 불필요 기능 제거, FF 전용으로 채움.
- **홈:** Scan → Pick → Play. 반유튜브(깔끔·여백·유한·액션강요0).
- **단계:** Phase 1 = 0(품질)+1(Pull엔진)+3(캡처). Phase 2 = Bridge(보류, 경계 미정).
- **법적:** 자막 스크래핑(`youtube-transcript`+`youtubei.js`)이 ToS 위반 = 제품화 최대 제약. 채널추가/메타데이터는 공식 API라 안전. 개인용은 리스크 낮음.

## 교훈
- 사용자의 "채널 추가 불편"은 표면 증상. 한 꺼풀 벗기니(왜 안 쓰나) 제품 정체성 문제가 본질이었음 → **요청을 액면가로 구현하지 말고 의도를 먼저 파라.**
- 제품화 가능성·법적 리스크는 기억으로 답하면 위험 → 현재 출처로 검증(워크플로 4축 리서치)했고, "자막 추출 방식"이 리스크 크기를 좌우함을 코드 확인 후 특정.
