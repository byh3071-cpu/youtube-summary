# Focus Feed 리디자인 브레인스토밍 (2026-06-24)

> 이 문서 = 2026-06-24 브레인스토밍 세션의 결론을 고정한 **재개 지점(resume point)**.
> 다음 단계는 이 방향으로 **Phase 1 스펙** 작성. 코드 세부는 `docs/PRD.md`·코드 우선.

## 0. 출발점 (사용자 원래 질문)

"포커스 피드에 유튜브 채널 추가가 채널 ID 복붙이라 불편 — 더 쉬운 방법?"
→ 파고드니 진짜 문제는 채널 추가가 아니라 **"FF를 안 쓰게 되는 이유"** 였음.

## 1. 한 줄 정체성 (요한 버전)

> **내 취향으로 끌리는 유튜브 영상을 가볍고 산뜻하게 띄워(발견) → 보고 싶게 만들고 → 마음에 든 영상을 부담 없이 내 지식 워크플로(NotebookLM·노션·아이디어뱅크)로 흘려보내는 중간 다리.**

"또 다른 유튜브 보는 곳"이 아님.

## 2. 핵심 통찰

- **왜 지금 FF를 안 쓰나:** 유용하지만 "끌리지 않음". 과거 **UI/UX 불만 + 잦은 버그**가 이탈 원인. "어느 정도 꾸미면 볼 듯" → **품질이 기능보다 선행 전제.**
- **습관은 이미 있다:** 그는 FF의 핵심 use-case(좌측 모니터에 유튜브 라디오처럼 틀기)를 **YouTube에서** 매일 함. FF는 *새 습관 창조*가 아니라 *기존 습관(좌측 모니터)을 뺏어오기*.
- **자연 흐름:** "스캔하겠지 → 골라보겠지 → 틀어놓겠지" = **Scan → Pick → Play**.
- **반(反)유튜브가 차별점:** 끌리는 앱(sort.to·NYT·Bluesky·vinyls)은 그리드·무한스크롤·썸네일 폭격이 아니라 *여백·세로 리스트·작은 단위·큰 액션 1개*로 차분함을 만듦.

## 3. 디자인 북극성 — "유튜브 × 스포티파이"

**유튜브(콘텐츠) × 스포티파이(껍데기)** 하이브리드:
내가 구독한 유튜브 채널을 **스포티파이처럼** 틀어놓고·큐에 담고·라이브러리로 관리. 영상이지만 음악앱처럼 조작 → 라디오 모드가 자연스러움. 단 **스포티파이를 FF에 최적화**(불필요 제거, FF 전용 기능으로 채움).

| 스포티파이에서 **제거**(FF 불필요) | FF 전용으로 **채움** |
|---|---|
| 음악·아티스트·앨범 개념 | 유튜브 채널 구독·피드 |
| 장르 캐러셀 폭격, 검색 중심 발견 | Scan→Pick→Play 피드(최신순/추천순) |
| 소셜(친구 활동)·협업 플레이리스트 | AI 후크 1줄·요약·끌림 랭킹 근거 |
| 광고·업셀 배너 | 채널 캡처(딸깍 추가)·(Phase2) 아이디어뱅크 브리지 |

**채택할 스포티파이 구조:** 좌측 사이드바 셸 · 하단 고정 플레이어바 · 라이브러리/큐 개념 · "재생"이 1급 시민 · 차분한 톤.

## 4. 홈 = Scan → Pick → Play

```

│  Focus Feed        [최신순 ▾ | 추천순]   [＋ 채널]   [🔍]  │  ← 정렬 토글 + 캡처
```

- **① 스캔:** 착지 = 내 채널 새 영상 깔끔 리스트. 끌림순(AI/개발 최신·새로움·깊이). 유한.
- **② 골라보기:** 카드 탭 → 집중 시청. `[▶]` 라디오 큐 담기 / `[⤓]` (Phase2) 아이디어뱅크로.
- **③ 틀어놓기:** `[▶ 전체 틀기]` → 좌측 모니터 BGM 스트림. 하단 플레이어바가 now-playing+큐.

## 5. 기능 구성

- **롱폼:** 큰 썸네일 세로 리스트 (CNN 히어로 + sort.to 깔끔).
- **숏폼:** 별도 가로 그리드 행 (Bluesky Video탭형). 풀스크린 스와이프 **강제 X** — 전용 모드는 원할 때만.
- **라디오:** 하단 고정 **플레이어바** (neuecast·Apple Music 미니바 + Spotify 셸). "전체 틀기" 0클릭(radio-garden). vinyls급 감각.
- **끌림 랭킹:** 명시(내 채널·카테고리) + 암묵(요약·북마크·오래본·라디오넣은) + 깊이추정. **왜 떴는지 투명.** 낚시·과장 ❌.
- **캡처:** `/add?url=` 딥링크 코어 1개 + 얇은 프론트(북마클릿 → 확장 → PWA 공유타깃). 채널추가=공식 API라 법적 안전.
- **원칙:** 깔끔·여백·유한·**액션강요 0**("내일 무조건 실행" 류 금지).

## 7. 법적 제약 (요약)

- **자막 AI요약 = 급소.** `youtube-transcript`+`youtubei.js` 스크래핑은 YouTube ToS 위반. 상세: 메모리 `focus-feed-transcript-tos-risk` / 워크플로 리서치 2026-06-24.
- **채널추가·메타데이터 = 안전** (공식 Data API v3).
- **개인용 단계 리스크 낮음.** 제품화하려면 자막을 클라이언트사이드(확장)로 + 짧은요약+원문링크 + 어트리뷰션.
