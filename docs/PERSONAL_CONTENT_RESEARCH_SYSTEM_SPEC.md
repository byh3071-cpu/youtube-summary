# Focus Feed 개인 콘텐츠 연구 시스템 스펙

## 1. 제품 결정

Focus Feed의 1차 사용자는 불특정 소비자가 아니라 **운영자인 나 자신**이다.

제품의 역할은 콘텐츠를 더 많이 추천하는 것이 아니다. 내가 직접 선택한 YouTube 채널과 RSS만 받아 보고, 필요한 콘텐츠를 깊게 이해하고, 근거와 맥락을 보존한 채 Notion에서 다시 활용할 수 있게 만드는 것이다.

> **Focus Feed는 알고리즘 없는 개인 콘텐츠 인박스이자, 영상과 글을 재사용 가능한 지식으로 바꾸는 개인 연구 시스템이다.**

### 핵심 가치

1. **통제권**: 플랫폼 추천 알고리즘 대신 내가 선택한 소스만 본다.
2. **집중**: 모든 콘텐츠를 소비하지 않고, 볼 가치가 있는 항목을 직접 선별한다.
3. **이해**: 트랜스크립트, 타임스탬프, 요약, 인용, 키워드로 원문을 깊게 파악한다.
4. **가공**: AI 결과와 내 메모를 결합해 2차 자료로 만든다.
5. **재사용**: Notion에 저장한 자료를 검색, 연결, 비교, 집필에 다시 사용한다.
6. **신뢰**: 원문과 AI 해석을 구분하고, 언제든 근거 구간으로 돌아갈 수 있다.

## 2. 하지 않을 일

다음 항목은 개인 연구 루프가 완성되기 전까지 핵심 우선순위가 아니다.

- 불특정 사용자를 위한 추천 알고리즘
- 행동을 과도하게 제안하는 범용 AI 액션 코치
- 팀 협업 기능 확장
- Stripe와 요금제 고도화
- 여러 생산성 서비스에 대한 동시 연동
- X, Reddit 등 소스 유형의 무제한 확장
- 완전 자동 게시, 자동 작업 생성, 자동 의사결정
- 벡터 DB나 지식 그래프 자체를 목적으로 한 인프라 확장

기존 구현은 제거하지 않되, 개인 연구 핵심 흐름을 방해하지 않는 보조 기능으로 유지한다.

## 3. 핵심 사용 시나리오

### 시나리오 A: 알고리즘 없는 일일 확인

1. Focus Feed를 연다.
2. 내가 등록한 YouTube와 RSS의 새 항목만 확인한다.
3. 제목, 출처, 길이, 발행일을 보고 빠르게 분류한다.
4. 관심 없는 항목은 제외하고, 나중에 볼 항목은 큐에 넣는다.
5. YouTube 홈이나 추천 피드를 열지 않고 확인을 끝낸다.

### 시나리오 B: 이동 중 의도적 소비

1. 처리 대기 큐에서 영상을 라디오에 넣는다.
2. 마지막 재생 위치부터 듣는다.
3. 중요한 구간에 타임스탬프 북마크나 짧은 메모를 남긴다.
4. 완료 후 `소비 완료` 또는 `추가 분석 필요`로 상태를 바꾼다.

### 시나리오 C: 영상 딥다이브

1. 중요한 영상에서 딥다이브를 실행한다.
2. 트랜스크립트와 섹션 구조를 확인한다.
3. 핵심 주장, 인용, 키워드, 인물, 제품, 질문을 추출한다.
4. AI 결과를 원문 타임스탬프와 대조한다.
5. 필요한 부분을 수정하거나 내 메모를 추가한다.

### 시나리오 D: Notion 2차 가공

1. 검토가 끝난 콘텐츠만 Notion으로 보낸다.
2. 같은 콘텐츠는 새 페이지를 중복 생성하지 않고 기존 페이지를 갱신한다.
3. 원문 정보, 트랜스크립트, AI 분석, 내 메모를 분리해 저장한다.
4. 여러 콘텐츠를 주제별로 묶어 비교 노트나 글감으로 재가공한다.

### 시나리오 E: 과거 지식 재발견

1. 주제나 질문으로 과거 콘텐츠를 찾는다.
2. 관련 영상, RSS, 인용, 내 메모를 함께 본다.
3. 서로 반복되거나 충돌하는 주장을 비교한다.
4. 과거 자료를 현재 프로젝트, 글, 의사결정에 다시 사용한다.

## 4. 제품의 핵심 루프

```text
선택한 소스 수집
  -> 새 콘텐츠 인박스
  -> 선별 및 처리 큐
  -> 시청·청취·읽기
  -> 트랜스크립트·딥다이브
  -> 사용자 검토·메모
  -> Notion 내보내기
  -> 검색·비교·재발견
  -> 소스와 관심사 조정
```

이 루프에 직접 기여하지 않는 기능은 후순위로 둔다.

## 5. 콘텐츠 상태 모델

콘텐츠는 명시적인 처리 상태를 가진다.

```text
inbox
  -> queued
  -> consuming
  -> consumed
  -> processing
  -> reviewed
  -> exported
  -> archived
```

별도 종료 상태:

- `dismissed`: 볼 가치가 없다고 판단
- `failed`: 수집, 분석, 내보내기 실패

### 상태의 의미

| 상태 | 의미 |
|---|---|
| `inbox` | 새로 수집됐지만 아직 판단하지 않음 |
| `queued` | 읽거나 들을 가치가 있다고 선택함 |
| `consuming` | 현재 소비 중이며 재생 위치나 읽기 진행률이 존재함 |
| `consumed` | 원문 소비가 끝남 |
| `processing` | 트랜스크립트 또는 AI 분석 생성 중 |
| `reviewed` | AI 결과를 사용자가 확인하고 필요한 메모를 추가함 |
| `exported` | Notion에 성공적으로 반영됨 |
| `archived` | 당장 활성 작업에는 쓰지 않지만 보존함 |

상태는 자동 추론보다 사용자의 명시적 선택을 우선한다.

## 6. 정보 구조

### Source

- 소스 ID와 유형
- 이름과 카테고리
- 활성 여부
- 마지막 동기화 시각
- 마지막 성공 시각
- 실패 이유
- 신규 콘텐츠 수
- 사용자가 지정한 우선순위

### Content Item

- 원문 ID, URL, 제목, 설명, 썸네일
- 소스, 작성자 또는 채널
- 발행일과 수집일
- 영상 길이와 콘텐츠 유형
- 처리 상태와 상태 변경 시각
- 재생 위치와 완료 여부
- 숨김, 보관, 큐 여부
- Notion 페이지 ID와 최종 동기화 시각

### Transcript

- 원문 언어
- 획득 방식
- 타임스탬프별 원문
- 획득 시각
- 실패 또는 품질 상태
- 원문 변경 또는 재수집 버전

### Research Artifact

- 한 문장 핵심
- 요약
- 섹션별 핵심 내용
- 주장과 근거 타임스탬프
- 직접 인용
- 키워드와 개념
- 인물, 제품, 조직
- 열린 질문
- 관련 콘텐츠
- AI 모델과 생성 시각
- 생성 버전

### User Annotation

- 사용자 메모
- 중요 구간
- 사용자 태그
- 내 프로젝트와의 관련성
- 후속 조사 질문
- 최종 검토 상태

AI 생성 정보와 사용자 작성 정보는 절대 같은 필드에 덮어쓰지 않는다.

## 7. 신뢰 원칙

### 근거 우선

- 모든 핵심 주장과 인용은 가능한 경우 타임스탬프를 가진다.
- AI가 만든 문장과 원문 인용을 시각적·데이터적으로 구분한다.
- 트랜스크립트가 없거나 품질이 낮으면 결과에 명확히 표시한다.
- 제목과 설명만 사용한 폴백 분석은 `degraded` 상태로 저장한다.

### 사용자 승인

- Notion 자동 전송보다 검토 후 전송을 기본값으로 한다.
- 자동 동기화는 사용자가 콘텐츠 유형별로 켤 때만 동작한다.
- AI가 만든 키워드, 인용, 액션은 사용자가 수정할 수 있어야 한다.

### 재현 가능성

- 분석 결과에 사용 모델, 프롬프트 버전, 생성 시각을 기록한다.
- 재생성해도 이전 결과를 즉시 삭제하지 않는다.
- Notion 내보내기는 동일 콘텐츠 ID를 기준으로 멱등하게 동작한다.

## 8. 기존 기능의 재배치

| 기존 기능 | 새 역할 |
|---|---|
| 통합 YouTube·RSS 피드 | 핵심: 개인 인박스 |
| 키워드·카테고리 필터 | 핵심: 인박스 선별 |
| 커스텀 소스 | 핵심: 알고리즘 통제 |
| 라디오 플레이어 | 핵심: 이동 중 소비 |
| 재생 위치·시청 기록 | 핵심: 소비 연속성 |
| 딥다이브 | 핵심: 연구 자료 생성 |
| 트랜스크립트 | 핵심: 근거 원문 |
| 요약·인사이트 | 핵심: 초기 가공 |
| Notion 동기화 | 핵심: 2차 가공과 보존 |
| 북마크·플레이리스트 | 핵심 보조: 처리 큐 |
| My Focus | 보조: 인박스 선별 기준 |
| Trend Radar | 보조: 선택한 소스 내부의 흐름 탐색 |
| Feed Q&A | 보조: 현재 인박스 탐색 |
| Todoist 링크 | 선택: 명시적 작업이 있을 때만 |
| 팀·Stripe | 보류: 개인 시스템 완성 후 판단 |

`My Focus`는 행동 코치가 아니라 **현재 관심사에 따라 인박스 우선순위를 조정하는 입력값**으로 제한한다.

## 9. Notion 저장 모델

### 원칙

- Focus Feed가 수집과 분석의 원본 시스템이다.
- Notion은 사람이 편집하고 조합하는 2차 작업 공간이다.
- Notion에서 모든 원본 데이터를 다시 읽어 동기화하려 하지 않는다.
- 초기에는 완전한 양방향 동기화보다 안정적인 단방향 upsert를 우선한다.

### 권장 페이지 구조

```text
기본 정보
- 제목, URL, 채널, 발행일, 길이
- Focus Feed 상태와 내부 ID

내 결론
- 내가 얻은 핵심
- 내 프로젝트와의 관련성
- 후속 조사 질문

AI 분석
- 한 문장 핵심
- 요약
- 주요 주장
- 키워드와 개념

근거
- 타임스탬프별 인용
- 섹션별 핵심

원문
- 트랜스크립트 또는 원문 링크

시스템 정보
- 분석 버전, 모델, 생성 시각, 최종 동기화 시각
```

### 동기화 상태

```text
not_requested
  -> queued
  -> syncing
  -> synced
  -> stale
  -> retrying
  -> failed
```

부분 실패 시 생성된 Notion 페이지 ID를 잃지 않아야 한다.

## 10. 검색과 재발견

검색은 다음 대상을 구분해 제공한다.

- 제목과 설명
- 트랜스크립트 원문
- AI 요약과 키워드
- 사용자 메모
- 인용과 타임스탬프
- 출처와 기간
- 처리 상태

### 재발견 기능 후보

1. 최근 저장했지만 다시 열지 않은 콘텐츠
2. 같은 개념을 반복해서 언급한 서로 다른 출처
3. 서로 반대되는 주장과 근거
4. 특정 프로젝트와 연결된 과거 자료
5. 지난달에는 드물었지만 이번 달 증가한 키워드
6. 메모는 남겼지만 Notion 가공이 끝나지 않은 콘텐츠

재발견은 새로운 추천이 아니라 **내가 이미 선택한 자료의 재사용**이다.

## 11. 단계별 개발 순서

### Phase 1: 개인 인박스

- 콘텐츠 상태 모델 도입
- 읽지 않음, 큐, 소비 완료, 제외, 보관
- 소스별 신규 개수와 동기화 상태
- 중복 제거와 안정적인 증분 수집
- 데스크톱·모바일 상태 동기화

완료 기준:

- YouTube 홈을 열지 않고 새 콘텐츠 확인과 선별을 끝낼 수 있다.
- 어느 기기에서 열어도 처리 상태가 동일하다.

### Phase 2: 소비 기록

- 재생 위치 서버 저장
- 타임스탬프 북마크와 짧은 메모
- 처리 큐와 이어 듣기
- 완료·중단·재방문 이력

완료 기준:

- 모바일에서 듣던 영상을 데스크톱에서 같은 위치부터 이어갈 수 있다.

### Phase 3: 연구 아티팩트

- 트랜스크립트 영속 저장과 품질 상태
- 딥다이브 결과 영속 저장
- AI 결과와 사용자 메모 분리
- 분석 버전과 재생성 이력
- 인용과 타임스탬프 검증

완료 기준:

- 같은 영상에 AI 비용을 반복 지불하지 않는다.
- 중요한 결과가 브라우저 캐시 삭제로 사라지지 않는다.

### Phase 4: Notion 파이프라인

- 검토 후 단방향 upsert
- 중복 페이지 방지
- 동기화 상태와 재시도
- 템플릿 버전 관리
- 사용자 메모와 AI 분석의 분리 저장

완료 기준:

- 동일 콘텐츠를 여러 번 전송해도 Notion 페이지가 하나만 유지된다.
- 실패 원인과 재시도 가능 여부를 알 수 있다.

### Phase 5: 검색과 재발견

- 통합 검색
- 관련 콘텐츠 연결
- 반복·충돌 주장 탐색
- 미완료 연구 큐
- 주간 재발견 브리핑

완료 기준:

- 과거 자료가 실제 글쓰기나 의사결정에 다시 사용된다.

## 12. 성공 지표

### 제품 사용 지표

- YouTube 홈을 열지 않고 보낸 연속 일수
- 새 콘텐츠 중 분류 완료 비율
- 큐에 넣은 콘텐츠의 소비 완료율
- 소비 완료 후 연구 자료로 가공한 비율
- Notion 내보내기 성공률
- 과거 콘텐츠 검색 또는 재사용 횟수
- 중복 분석으로 낭비된 AI 호출 수

### 개인 효용 지표

- 하루 콘텐츠 탐색에 쓰는 시간
- 실제로 유용했다고 판단한 콘텐츠 수
- 콘텐츠에서 생성된 메모, 글, 의사결정 수
- 저장 후 다시 사용하지 않은 콘텐츠 비율

기능 개수, 총 피드 수, AI 호출 횟수는 핵심 성공 지표가 아니다.

## 13. 추가 아이디어

### Research Session

여러 콘텐츠를 하나의 조사 주제로 묶는다.

예:

- Claude Code 에이전트 운영
- 1인 SaaS 온보딩
- AI 영상 요약 제품 비교

세션에는 질문, 선택한 콘텐츠, 인용, 내 결론, 미해결 질문이 포함된다. 단순 태그보다 목적이 명확하며, 최종적으로 Notion의 비교 노트나 글 초안이 된다.

### Claim Ledger

영상과 글에서 나온 핵심 주장을 별도 단위로 저장한다.

- 주장
- 출처
- 근거 구간
- 내 판단
- 지지하거나 반박하는 다른 콘텐츠

지식 그래프를 먼저 만들지 않고도 출처 간 비교와 검증을 가능하게 한다.

### Evidence Pack

글이나 의사결정에 사용할 자료를 묶어 내보낸다.

- 선택한 인용
- 타임스탬프 링크
- 핵심 요약
- 내 메모
- 출처 목록

Notion 페이지 전체를 보내는 것보다 실제 집필과 조사에 직접 유용하다.

### Source Health

소스별 가치와 건강 상태를 보여준다.

- 최근 30일 발행 수
- 내가 실제 소비한 비율
- 유용한 연구 자료로 전환된 비율
- 반복적으로 제외한 비율
- 수집 실패 상태

이를 통해 구독 소스를 계속 정리하고 알고리즘 없는 피드의 품질을 유지한다.

### Research Debt

다음 상태를 연구 부채로 모아 보여준다.

- 소비했지만 메모하지 않은 중요 콘텐츠
- 딥다이브했지만 검토하지 않은 콘텐츠
- 검토했지만 Notion에 보내지 않은 콘텐츠
- 동기화 실패 콘텐츠

일반적인 AI 액션 추천보다 현재 워크플로의 미완료 단계를 알려주는 것이 지속적으로 유용하다.

## 14. 다음 제품 결정

구현 전 다음 원칙을 확정한다.

1. Focus Feed와 Notion 중 어느 쪽이 어떤 데이터의 원본인가?
2. 트랜스크립트 전체를 Supabase에 저장할 것인가, 외부 저장소를 사용할 것인가?
3. AI 분석 버전을 몇 개까지 보존할 것인가?
4. 콘텐츠 상태 변경을 얼마나 자동화할 것인가?
5. Notion 자동 전송은 기본 비활성으로 둘 것인가?
6. 검색 범위에 트랜스크립트 전체를 포함할 것인가?
7. 개인용 시스템에서 데이터 보존 기간과 삭제 정책은 무엇인가?

권장 기본값:

- 수집·상태·분석 원본은 Focus Feed
- 편집·조합·집필은 Notion
- 자동화보다 명시적 사용자 승인
- 트랜스크립트와 AI 분석은 영속 저장
- 삭제보다 보관, 단 사용자가 완전 삭제 가능
- 개인 핵심 루프 완성 전 팀·결제 기능 확장 중단

## 15. 확장과 고도화 원칙

확장은 기능 수를 늘리는 작업이 아니다. 현재의 연구 루프가 더 많은 자료를 안정적으로 처리하거나, 더 깊게 이해하거나, 과거 지식을 더 쉽게 재사용하도록 만드는 작업이다.

새 기능은 다음 질문 중 하나에 명확히 답해야 한다.

1. 알고리즘 플랫폼을 방문하지 않아도 되는가?
2. 중요한 콘텐츠를 더 정확히 선별할 수 있는가?
3. 원문을 더 깊고 신뢰성 있게 이해할 수 있는가?
4. 여러 콘텐츠 사이의 관계를 발견할 수 있는가?
5. 가공된 자료를 실제 글과 의사결정에 다시 사용할 수 있는가?
6. 반복 비용과 수작업을 줄이면서도 사용자의 통제권을 유지하는가?

어느 질문에도 답하지 못하는 기능은 확장이 아니라 복잡성이다.

### 도입 게이트

새 기능은 다음 조건을 만족할 때만 핵심 로드맵에 넣는다.

- 기존 핵심 루프의 단계를 하나 이상 줄인다.
- 결과물에 원문 출처나 생성 이력이 남는다.
- 실패해도 기존 데이터가 손상되지 않는다.
- 자동화는 사용자가 끄거나 되돌릴 수 있다.
- 저장 공간, AI 호출, 외부 API 비용을 확인할 수 있다.
- 특정 공급자나 Notion에 데이터가 잠기지 않는다.

## 16. 고도화 축

### A. 수집 고도화

초기에는 YouTube와 RSS의 품질을 높이는 데 집중한다.

- 마지막 수집 지점 기반 증분 동기화
- 삭제, 비공개, 재업로드 콘텐츠 처리
- 동일 URL과 유사 제목 중복 제거
- 채널별 수집 주기와 일시 중지
- 소스별 실패 원인과 재시도
- 발행량이 너무 많은 소스의 일일 상한
- Shorts, 라이브, 긴 영상에 대한 소스별 규칙

핵심이 안정된 뒤 추가할 수 있는 소스:

- 팟캐스트 RSS
- 이메일 뉴스레터 전달 주소
- 사용자가 직접 저장한 웹 문서
- PDF와 공개 리포트
- GitHub Release와 공식 문서 변경 피드

새 소스는 `원문 주소`, `발행 정보`, `본문 또는 트랜스크립트`, `안정적인 외부 ID`를 제공할 수 있을 때만 도입한다.

### B. 이해 고도화

단일 콘텐츠 요약을 넘어 연구에 필요한 구조를 만든다.

- 주장과 근거 분리
- 사실, 의견, 예측, 광고성 문장 분류
- 핵심 개념과 용어 정의
- 인물, 회사, 제품, 논문, 도구 추출
- 수치와 날짜 추출
- 출처가 제시된 주장과 출처 없는 주장 구분
- 영상 내 관점 변화와 결론 구조 파악
- AI 확신도보다 근거 품질 상태 표시

권장 품질 상태:

```text
source_verified
transcript_verified
transcript_partial
metadata_only
ai_unreviewed
user_reviewed
```

### C. 비교 연구 지능

Focus Feed가 일반 요약 앱과 가장 크게 달라질 수 있는 영역이다.

#### Topic Timeline

특정 주제가 시간에 따라 어떻게 변했는지 보여준다.

- 최초 등장
- 반복 언급
- 중요한 발표
- 주장의 변화
- 제품 또는 모델 버전 변화

#### Claim Comparison

동일 주제에 대한 여러 출처의 주장을 나란히 비교한다.

```text
주장
  -> 지지하는 출처
  -> 반박하는 출처
  -> 근거 구간
  -> 내 판단
  -> 아직 확인되지 않은 부분
```

#### Novelty Detection

새 콘텐츠가 기존 라이브러리에 비해 무엇을 새롭게 말하는지 표시한다.

- 기존 자료의 반복
- 새로운 근거
- 기존 주장 수정
- 반대 관점
- 새로운 도구나 사례

이 기능은 단순 중요도 점수보다 훨씬 지속적인 가치가 있다.

#### Contradiction Queue

서로 충돌하는 주장을 자동으로 확정하지 않고 사용자 검토 큐로 보낸다. AI는 충돌 후보와 근거만 제시하고 최종 판단은 사용자가 한다.

### D. 연구 작업 공간

#### Research Session 고도화

조사 세션을 단순 폴더가 아닌 명시적 연구 단위로 만든다.

- 연구 질문
- 범위와 제외 조건
- 선택한 콘텐츠
- 핵심 주장
- 증거 묶음
- 잠정 결론
- 미해결 질문
- 최종 산출물 링크

#### Working Note

Notion으로 보내기 전 Focus Feed 안에서 임시 연구 노트를 만든다.

- 콘텐츠를 끌어다 놓아 배치
- 인용과 타임스탬프 삽입
- 내 문장과 AI 초안 구분
- 사용한 근거 목록 자동 생성

#### Output Templates

같은 자료를 목적에 따라 다르게 가공한다.

- 비교 분석
- 제품 조사
- 기술 학습 노트
- 의사결정 메모
- 블로그 글 초안
- 영상 또는 뉴스레터 기획안

템플릿은 원본 데이터를 변경하지 않고 출력 구조만 바꾼다.

### E. 제한된 자동화

범용 액션 추천 대신 명확하고 되돌릴 수 있는 규칙 자동화를 사용한다.

예:

- 특정 채널의 긴 영상은 자동으로 처리 큐에 추가
- 90% 이상 재생하면 `consumed`로 전환
- 사용자가 중요 표시한 콘텐츠만 트랜스크립트 처리
- 검토 완료 콘텐츠만 Notion 동기화 대기열에 추가
- 동기화 실패 시 최대 횟수 내에서 재시도
- 30일간 활용되지 않은 소스는 정리 후보로 표시

금지하는 자동화:

- 검토하지 않은 AI 결과의 자동 외부 게시
- 모든 새 콘텐츠의 자동 AI 분석
- 사용자 승인 없는 작업 생성
- 실패를 숨기고 계속 진행하는 자동 재시도
- 원본이나 사용자 메모를 AI 결과로 덮어쓰기

### F. 검색 고도화

검색은 단계적으로 확장한다.

1. 메타데이터와 사용자 태그 검색
2. 트랜스크립트 전문 검색
3. 의미 기반 검색
4. 주장과 인용 단위 검색
5. 연구 세션과 프로젝트 문맥 검색

검색 결과는 항상 왜 매칭됐는지 보여준다.

- 제목 일치
- 트랜스크립트 구간
- 사용자 메모
- AI 키워드
- 관련 주장

의미 검색을 도입하더라도 일반 텍스트 검색을 대체하지 않는다.

### G. 개인화 고도화

개인화는 클릭을 늘리는 추천이 아니라 연구 적합도를 높이는 방향으로 사용한다.

- 현재 프로젝트
- 장기 관심 분야
- 제외할 주제
- 선호하는 출처
- 선호하는 콘텐츠 길이
- 이미 충분히 알고 있는 개념
- 새롭게 학습하려는 개념

추천 결과에는 점수보다 이유를 제공한다.

```text
현재 프로젝트와 직접 관련
기존 자료와 반대 관점
새로운 근거 포함
이미 본 내용과 대부분 중복
```

### H. 비용과 모델 고도화

모든 작업에 가장 비싼 모델을 쓰지 않는다.

- 규칙 기반 전처리
- 저비용 모델로 분류와 키워드 추출
- 고품질 모델은 중요 콘텐츠의 깊은 분석에만 사용
- 긴 트랜스크립트는 섹션 캐시 재사용
- 동일 콘텐츠의 동시 분석 요청 병합
- 프롬프트와 결과 해시 기반 중복 호출 방지
- 콘텐츠별 예상 비용과 실제 비용 기록

사용자는 `빠른 분석`, `표준 분석`, `깊은 연구` 수준을 선택할 수 있다.

### I. 데이터 독립성과 장기 보존

개인 시스템은 서비스나 공급자가 바뀌어도 데이터가 남아야 한다.

- 전체 자료 JSON 내보내기
- Markdown과 원문 파일 내보내기
- Notion 없이도 복원 가능한 백업
- 트랜스크립트와 사용자 메모의 정기 백업
- 첨부 파일 저장 위치 선택
- 삭제와 보존 기간 설정
- AI 모델과 외부 서비스 교체 가능한 어댑터 구조

장기적으로는 로컬 우선 저장 또는 개인 저장소 연동도 검토한다.
